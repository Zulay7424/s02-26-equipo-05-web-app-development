# QA Manual Testing Project - No Country Simulation

##  Perfil del Proyecto
Este repositorio contiene la documentación técnica, el plan de pruebas y la ejecución de casos de prueba realizados durante mi participación en la **Simulación Laboral de No Country (Cohorte Febrero 2026)**. Como **Ingeniera y QA Analyst**, mi objetivo principal fue garantizar la calidad, fiabilidad y usabilidad de la aplicación mediante metodologías de prueba rigurosas.

##  Rol y Responsabilidades
En este proyecto, asumí la responsabilidad total del ciclo de vida de las pruebas (**STLC**), asegurando que cada funcionalidad cumpliera con los requisitos del negocio. Mis tareas incluyeron:
* **Análisis de Requerimientos:** Revisión detallada de las especificaciones para asegurar una cobertura de prueba del 100%.
* **Diseño de Casos de Prueba:** Creación de escenarios de prueba detallados para flujos críticos y secundarios.
* **Ejecución de Pruebas Manuales:** Validación de funcionalidad, usabilidad y rendimiento.
* **Gestión de Defectos:** Identificación, documentación y seguimiento de bugs utilizando herramientas de tracking.
* **Colaboración Ágil:** Trabajo estrecho con desarrolladores y product managers en un entorno remoto internacional.

##  Tipos de Pruebas Realizadas
Para asegurar la robustez del software, apliqué las siguientes estrategias:
* **Smoke Testing:** Verificación rápida de las funciones principales tras cada despliegue.
* **Regression Testing:** Pruebas exhaustivas para asegurar que los nuevos cambios no afectaran funcionalidades existentes.
* **Exploratory Testing:** Sesiones de prueba libres para identificar comportamientos inesperados desde la perspectiva del usuario.
* **Usability Testing:** Evaluación de la interfaz para garantizar una experiencia de usuario fluida.

##  Documentación de QA
Puedes encontrar los entregables técnicos en la carpeta `/QA-Documentation` (o el nombre que elijas), los cuales incluyen:
1.  **Test Plan:** Estrategia y alcance de las pruebas.
2.  **Test Cases Suite:** Documento con pasos, resultados esperados y estados (Pass/Fail).
3.  **Bug Reports:** Informes detallados con capturas de pantalla y pasos para reproducir errores encontrados.
